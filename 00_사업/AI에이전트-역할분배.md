# AI 에이전트 역할 분배 체계

> AEON의 Top-Down 조직 원칙에 따라, 각 에이전트의 역할과 경계를 명확히 정의합니다.
> 역할이 명확해야 혼란 없이 사업이 깊어집니다.

---

## 현재 단계 (Phase 0: 창업 초기)

```
창업자 (배명규)
    └── Claude Code (전 영역 보조)
         ├── 기획 · 전략
         ├── 문서 작성
         ├── 영업 자료
         ├── 기술 구현
         └── 지식베이스 구축
```

> 지금은 Claude Code가 만능이지만, 역할이 분산될 것을 항상 염두에 두고 작업한다.
> 각 파일·폴더 구조가 곧 미래 에이전트의 "업무 영역"이 된다.

---

## 목표 단계 (Phase 3+: 에이전트 분화)

### 에이전트 전체 맵

```
창업자 (최종 의사결정)
    │
    ├── [전략 레이어]
    │    └── 선행개발 전담 에이전트 (R&D · 미래 기획)
    │
    ├── [비즈니스 레이어]
    │    ├── 영업 전담 에이전트
    │    ├── 마케팅 전담 에이전트
    │    └── 재무기획 전담 에이전트
    │
    ├── [운영 레이어]
    │    ├── 회계 전담 에이전트
    │    ├── 인사 전담 에이전트
    │    └── 반복업무 전담 에이전트
    │
    └── [기술 레이어]
         ├── 개발 전담 에이전트
         └── 디자인 전담 에이전트 (UI/UX · Figma)
```

## 에이전트 협업 프로토콜

### 정보 전달 흐름
```
창업자 → (자연어 지시) → 담당 에이전트
담당 에이전트 → (결과물) → 해당 폴더 MD 파일
다른 에이전트 → (참조) → 해당 MD 파일 읽기
```

### 충돌 방지 원칙
1. 각 에이전트는 **자신의 폴더**만 수정한다
2. 다른 에이전트의 파일을 **참조**는 하되 **직접 수정 금지**
3. 수정이 필요하면 창업자에게 보고 후 진행
4. `CLAUDE.md`는 창업자만 업데이트한다

### 현재 Claude Code의 역할
모든 에이전트 역할을 단독으로 수행하되,
각 작업 결과를 **해당 에이전트의 폴더에 저장**하는 습관을 지금부터 들인다.
→ 나중에 실제 에이전트 분화 시 마이그레이션 비용 최소화.
