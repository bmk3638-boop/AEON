## 성공한 최종 코드 핵심 3단계

### Step 1. Chrome 디버그 모드 실행 (수동 1회)
```powershell
& "C:\Program Files\Google\Chrome\Application\chrome.exe" `
  --remote-debugging-port=9222 `
  --user-data-dir="C:\chrome-debug"
```

### Step 2. 기업회원 탭 클릭 — XPath 텍스트 매칭
```python
corp_tab = wait.until(EC.presence_of_element_located(
    (By.XPATH,
     "//*[contains(text(),'기업회원') and (self::a or self::li or self::button or self::span)]")
))
driver.execute_script("arguments[0].click();", corp_tab)
time.sleep(1.5)
```
- 탭 요소: `<button class="btn_tab t_com">`
- CSS `:contains()` 는 작동 안 함 → XPath 텍스트 매칭으로 해결

### Step 3. Visible 기준 입력 필드 선택
```python
id_field = driver.execute_script("""
    var els = Array.from(document.querySelectorAll('input[type="text"]'))
        .filter(el => el.offsetParent !== null && !el.disabled);
    return els[els.length - 1];
""")
```
- 탭 클릭 후 기업회원 폼이 활성화되면 동일한 `#id` `#password` 필드를 공유
- 탭 상태만 달라지므로 탭 클릭이 핵심

## 발견된 사실 (중요)

1. **사람인 로그인 폼 구조**: 개인회원/기업회원 탭이 같은 `#id`, `#password` 필드를 공유함
2. **탭 클릭만이 핵심**: 탭 상태(어떤 탭이 active인지)가 서버로 전달되어 로그인 경로가 갈림
