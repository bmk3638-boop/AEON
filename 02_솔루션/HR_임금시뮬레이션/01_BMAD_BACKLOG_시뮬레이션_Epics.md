# WageSim BMAD 백로그 — 시뮬레이션 10 Epics

> 각 Epic = "단일 질문 엔진" 1개. 4계층(L1 콕핏 / L2 시나리오엔진 / L3 도메인·룰 / L4 데이터·수집) 위에 얹는다.
> 공통 NFR: ① 개인 레코드 0건 업로드(L4 로컬 익명화) ② What-if만(자동 적용 없음, 원칙6) ③ 검증 게이트(3개월 급여 재계산 대조).

---

## 플랫폼 Epic (Epic 1에 포함되어 함께 구축 — 재사용 기반)

- **P-A. L4 로컬 전처리기 v0** — 급여원장 → 직군×연령×호봉 분포 + 수당 합계. 개인 레코드 미반출 보장.
- **P-B. 수당 분류 룰셋 v0** — 수당의 통상임금 포함 여부 입력/판단(고객 입력형, 자격 취득 후 검증 결합).
- **P-C. 콕핏 셸** — 레버·차트·시나리오 A/B·리포트 출력 프레임 (데모 HTML 재사용).
- **P-D. 검증 체크리스트** — 표본 3개월 재계산 → 실지급액 대조.

---

## Tier A — 리스크 금액 (먼저)

### Epic 1. 통상임금 재산정 ★ v1
- **질문**: 정기상여 통상임금 포함 시 연 인건비 증가분 + 과거 3년 소급 부채?
- **출력**: 연 +N억 / 소급 부채 N억 / 영향 인원
- **수용기준(초안)**:
  - Given 분류표에서 정기상여를 통상임금 포함으로 설정, when 시나리오 실행, then 통상임금·연장수당 재산정 → 연 증가분과 3년 소급액을 금액으로 출력.
  - Given 분포 데이터만 있고 개인 레코드 없음, then 결과는 분포 기반 추정이며 추정 범위(±)를 표시.
  - 검증: 표본 직군 3개월 재계산이 실지급액과 오차 ±N% 이내.
- **플랫폼**: P-A·B·C·D 전부 함께 구축(이 Epic이 플랫폼의 70%).

### Epic 2. 정년 연장 인건비
- 질문: 60→63→65 단계 적용 시 10년 인건비 곡선?
- 출력: 10년 누적 차액 / 연도별 곡선. (콕핏 정년 레버 재사용)

### Epic 3. 퇴직금 충당부채
- 질문: 정년연장·인상 시 DB형 퇴직급여 충당부채 증가분?
- 출력: 충당부채 증가분(재무제표 임팩트). Epic 2와 근속·정년 데이터 공유.

---

## Tier B — 정책 대응 (다음)

### Epic 4. 최저임금 인상 영향
- 질문: 내년 최저임금 인상 시 영향 인원·금액(통상임금 산입 연동)?
- 출력: 영향 인원 / 추가 인건비. (시장 가장 넓음 — 매년 반복)

### Epic 5. 주4일제 / 근로시간 단축
- 질문: 주4일제(또는 40→36h) 시 인건비·추가 채용 필요 인원?
- 출력: 인건비 변동 / 추가 채용 N명.

### Epic 6. 임금피크제
- 질문: 56세부터 연 X% 감액 시 절감액 vs 고령자고용법 리스크?
- 출력: 절감액 / 리스크 플래그. (콕핏 피크 레버 재사용)

---

## Tier C — 구조 전환 (그 다음)

### Epic 7. 호봉제 → 직무급 전환
- 질문: 직무급 전환 시 총인건비 변동 + 승자/패자 분포?
- 출력: 총인건비 변동 / 직원 증감 히트맵. (HR OS 직무·보상과 직결)

### Epic 8. 베이스업 / 임금협상 시나리오
- 질문: 연 X% 인상 누적 + 노사 협상안 A/B 비교?
- 출력: 시나리오 비교 차트(협상 테이블용).

---

## Tier D — 인력 구조 (마지막)

### Epic 9. 인력 고령화 + 정년 결원 충원
- 질문: 10년 후 연령 분포? 퇴직→신입 충원 시 절감?
- 출력: 연령 곡선 / 충원율별 비용. (콕핏 충원 레버 재사용)

### Epic 10. 이직률 / 적정 채용 시나리오
- 질문: 이직률 X% 변화 시 채용비용·생산성 손실? 적정 채용 규모?
- 출력: 이직 비용 / 권장 채용 N명. (HR OS 이직·확보 모듈 결합 지점)
