## 1. 기능과 목적 — 파트별 전수

> HR 라이프사이클(확보→재직→성장→보상→관계→이직) 순. 각 파트 = 실제 라우터/페이지.

### 1-1. 확보(채용) OS
- **목적**: 채용계획→공고→지원자→면접→최종합격→입사전환을 하나의 파이프라인으로.
- **기능**: 채용의뢰/워크포스 요청, 공고 관리, **사람인 자동 수집·이력서 PDF 일괄 다운로드(실행 기능, 비타협 항목)**, 지원자 파싱·분류, 면접 일정, 오퍼 승인, 건강검진, 온보딩, 최종합격→예정사원 전환.
- **라우터**: `acquisition`, `acquisition_v2`, `recruiting`, `recruitment`, `applicants`, `interviews`, `workforce`, `recruiting_agent`.
- **페이지**: `AcqRecruiting/Candidates/Interviews/OfferApproval/FinalHire/Onboarding/MedicalChecks/Resumes` 등 20+.
- **LLM**: 지원자 분석·평가 초안(`acquisition`, `recruitment`, `recruiting_agent`가 Gemini 호출).

### 1-2. 조직·발령 OS
- **목적**: 사원번호 기반 계정·조직도·발령(전보/승진/휴직/복직/파견)을 원천 관리.
- **기능**: 직원 마스터(입사·계정 연결), 조직도/정원·현원, **발령 기안→승인→조직·근태·급여 자동 반영**(Synapse #5/7/8), 발령 인텔리전스.
- **라우터**: `employees`, `hr_appointments`, `hr_events`.
- **페이지**: `OrganizationMgmt`, `EmployeeList/Profile`.
- **LLM**: 발령 영향분석 초안(`hr_appointments` generate_json).

### 1-3. 직무·스킬 OS
- **목적**: JD·직무등급·역량밴드·스킬 라이브러리·평가기준 연결.
- **기능**: 직무관리(JD 버전), 직무 아키텍처, 스킬 라이브러리, 인재(talent) 스킬 추출·매칭.
- **라우터**: `jobs`, `skills`, `talent`.
- **페이지**: `JobMgmt`, `JobArchitecture`, `SkillLibrary`.
- **LLM**: **talent가 가장 LLM 집약적** — 스킬 추출/정규화(`generate_json`), JD 동기화(`jobs`).

### 1-4. 근태 OS
- **목적**: 출퇴근·근무시간·휴가·연장/야간/휴일·유연근무·법정 리스크.
- **기능**: 근태 총대시보드, 휴가(16종 코드), 근무상태(코드), **주52시간 리스크 감지→노무 전파**(Synapse #19), 연차 만료 촉진, 근태 자율운영(슬롯/이벤트, Story 240~) .
- **라우터**: `attendance`, `attendance_autonomy`.
- **페이지**: `Attendance`, `AttendanceAutonomy`.

### 1-5. 노무 OS
- **목적**: 컴플라이언스 핵심 — 사건·징계·괴롭힘·노사협의회·고충·노동청 대응.
- **기능**: 노무 사건 칸반, 징계(사유/양정 코드), 괴롭힘 조사·보호조치, 노사협의회 법정시트, 복합 리스크 신호(`/labor/composite-risk`, Synapse #20).
- **라우터**: `labor`. **페이지**: `LaborOS`.

### 1-6. 평가 OS
- **목적**: KPI/OKR·평가·보정·이의신청·포상 후보.
- **기능**: 평가 사이클(코드), 평가등급(S~C), 보정, **성과→보상 후보 연계**(Synapse #11), 성과갭→교육/직무 역방향 제안(R2/R3).
- **라우터**: `performance`. **페이지**: `PerformanceMgmt`, `EvaluationForm`.

### 1-7. 보상 OS
- **목적**: 급여·수당·성과급·포상·임금밴드·공정성.
- **기능**: 수당 유형(코드), 급여마감 상태, 호봉/직급, 급여형태(직군별 호봉/연봉), 발령·평가→지급대기 원장 생성(비파괴).
- **라우터**: `reward`. **페이지**: `RewardOS`.

### 1-8. 교육·경력 OS
- **목적**: 교육 신청·법정교육 이수·경력개발·출장.
- **기능**: 교육 유형(코드), **교육 승인→이수이력+근태부재 자동 등록**(Synapse #15/16), 출장 승인→근태+정산(#23/24), IDP 스킬수준.
- **라우터**: `growth`. **페이지**: (HR OS growth 경로).

### 1-9. 유지(관계) OS
- **목적**: 1on1·몰입도(Gallup Q12)·eNPS·복리후생.
- **기능**: 설문 유형(코드), **Q12 급락→이직위험 신호**(Synapse #9), 이직위험→1on1 역방향(R1), 미팅 상태.
- **라우터**: `relations`. **페이지**: `RelationsOS`.

### 1-10. 이직 OS
- **목적**: 퇴직 신청·인수인계·퇴직금·계정 비활성·이직위험.
- **기능**: 퇴직유형/상태(코드), 퇴직금 계산, 계정 비활성 승인, 이직위험(`/relations/risk-followup`).
- **라우터**: `separation`. **페이지**: `SeparationOS`.

### 1-11. 라이프케어 OS
- **목적**: 건강검진·모성보호·식수(구내식당).
- **기능**: 검진유형/모성보호 프로그램(코드), 법정조치 기한 관리.
- **라우터**: `lifecare`. **페이지**: `LifeCareOS`.

### 1-12. 증명·컴플라이언스 센터
- **목적**: HR 증명서 발급·검증·폐기, 법정기한·감사·인증.
- **기능**: 증명서(재직/경력/원천), **AI 증명서 초안**(`hr_certificates` generate_json), 인증/허가, 체크리스트 분류, 법정기한 알림, `compliance_agent`.
- **라우터**: `hr_certificates`, `compliance`, `compliance_agent`. **페이지**: `ComplianceCenter`.

### 1-13. HR Analytics
- **목적**: 운영 건강도·리스크 Top5·인력/채용/성과/이직 지표.
- **라우터**: `analytics`, `report_agent`(리포트 생성). **페이지**: `HRAnalytics`.

### 1-14. HR Synapse / Agent (엔진)

- **라우터**: `hr_events`, `agent_actions`, `context_radar`. **페이지**: `HRSynapseHub/Summary/Map`, `AgentActions`, `AgentWorkbench`, `ContextRadar`.

### 1-15. 플랫폼(설계자) 계층

## 3. 모듈 연결 구조 (핵심 질문)

### 3-1. 코드로 연결 (Source of Truth 단일화)
- "유형/상태" 값은 자유문자열 금지 → `(code_group, code)` 공통코드. 시냅스·원장·감사가 **라벨이 아니라 코드**로 연결 → 표기가 흔들려도 안 깨짐.
- 예: 발령 승인 → `organization.appointment_type:140`가 `SynapseRun.code_keys`에 실려 근태로 흐름(실증 완료).

### 3-2. 프론트↔백엔드
- REST(JSON) + JWT Bearer(localStorage). `VITE_API_URL` 베이스. 
- 프론트 상태: `AuthContext`(로그인/역할), `PermissionsContext`(`/api/v1/auth/me/permissions` 읽어 페이지 게이팅).
- 대시보드/칸반은 클릭→상세 이동(제품 원칙 1·2).

### 3-3. 의도 색인검색 (Context Radar)
- `scripts/context_radar.py`가 docs/knowledge·bmad·프론트페이지·백엔드 라우터를 인덱싱(`SOURCE_GLOBS`).
- **렉시컬 스코어링 + 한글 시놉시스맵**(예: "최종합격"→final/finalpassed…, "중복"→duplicate/reuse…) = 경량 온톨로지. **벡터 임베딩 아님**(정직히 표기).
- 용도: (a) 개발 시 관련 파일/흐름 탐색, (b) 챗봇 근거 주입.

### 3-4. 챗봇 / LLM 연결
- `claude_ai.chat()` = **Gemini** 멀티턴. 시스템프롬프트(HR 7년 컨설턴트 페르소나) + 실시간 컨텍스트(채용현황·HR통계) + **Context Radar 근거 주입** = 경량 RAG.
- `generate()`/`generate_json()` = 모듈들이 초안·추출에 호출. **키 없으면 fallback**(데모 안전).
- LLM 사용 모듈: talent(스킬), acquisition/recruitment/recruiting_agent(채용), hr_certificates(증명서), hr_appointments(발령분석), jobs(JD), compliance_agent, report_agent.

### 3-5. 주요 모듈 스킬 (AI 에이전트)
- `recruiting_agent`(069), `compliance_agent`(070), `report_agent`(071) — v1. Gemini `generate` 기반 초안 생성.
- `talent` 스킬 서비스 — 스킬 추출/정규화(LLM+규칙 혼합).
- **Agent Harness(안전 툴콜·승인·피드백루프)**는 **[설계]** — 현재는 Agent Action 큐가 그 역할의 실구현 부분.

### 3-6. 온톨로지·지식그래프 (정직한 현황)
- **[설계]**: Ontology(정본 엔티티/관계), Knowledge Graph(Postgres-first→필요시 Neo4j), GraphRAG(승인된 그래프 사실 검색). 1차 범위=Appointment Intelligence Graph.
- **현재 실구현**: 공통코드 온톨로지(용어·코드) + 시냅스 연결표 + Context Radar 시놉시스맵이 "온톨로지의 씨앗". 완전한 그래프 검색은 미구현.

## 6. 타사 시스템에 녹아드는(종속) 방안

> 목표: 경동나비엔 같은 회사의 **기존 ERP/그룹웨어(더존·SAP·자체)에 AEON을 얹거나 데이터로 연결**.

### 6-1. 연결 4단계(난이도 순)
1. **읽기 연동(Read-only)**: 고객 인사DB를 CSV/Excel/API로 **가져와** AEON에서 분석·제안만. 위험 낮음, 도입 빠름 → **파일럿 1단계 권장**.
2. **양방향 API 연동**: 표준 REST + 웹훅. 공통코드 매핑표로 고객 코드↔AEON 코드 변환. 발령·근태 결과를 고객 시스템에 되쓰기.
3. **임베드(위젯/iframe/SSO)**: AEON 화면(예: 발령 승인·리스크 대시보드)을 고객 포털에 삽입. SSO(SAML/OIDC)로 계정 통합.
4. **엔진 종속(백엔드 서비스)**: AEON의 Synapse·Agent·QA를 고객 시스템 뒤에서 엔진으로 제공(헤드리스). 고객 UI 유지, AEON은 두뇌.

### 6-2. 종속을 위한 기술 작업
- **커넥터 계층**: 고객 시스템별 어댑터(더존/SAP/그룹웨어). MCP_INTEGRATION_BACKLOG의 커넥터 원칙(계정·권한·보안·데이터 검증) 준수.
- **코드 매핑 거버넌스**: 고객의 자유문자열 → AEON 공통코드 매핑표(리졸버·별칭 재사용). 이게 종속의 핵심 접착제.
- **멀티테넌트 격리**: 테넌트별 데이터·기능 on/off(이미 platform_lab에 존재) 강화.
- **인증 통합**: SSO(OIDC/SAML) + 역할 매핑(고객 직급→3모드).
- **데이터 계약**: PII 경계·마스킹·외부 LLM 전송 금지 정책(보안 체크리스트).
- **감사·컴플라이언스**: AEON AuditLog를 고객 감사에 제공 → "우리 시스템 얹었더니 감사 대응이 쉬워졌다"가 lock-in 포인트.

### 6-3. 종속(lock-in) 전략
- **데이터 중력**: 고객 데이터가 AEON 코드·시냅스로 정리될수록 이탈 비용↑. 단, "데이터는 고객 자산"(원칙5) → 이관 가이드는 항상 제공(신뢰가 곧 lock-in).
- **워크플로 종속**: 승인 게이트·감사가 고객 업무 프로세스에 박히면 대체 곤란.
- **법령 최신성 서비스**: 정기 법령 반영 리테이너 = 지속 종속.
