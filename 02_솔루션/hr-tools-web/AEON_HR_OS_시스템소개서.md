# AEON HR OS 시스템 소개서

## 엘리베이터 피치

> **AEON HR OS는 채용부터 이직까지 HR의 모든 업무를, 흩어진 엑셀이 아니라 하나의 데이터 흐름으로 잇는 HR 운영체제(Operating System)입니다.**
> 한 곳에 입력하면 관련 업무로 자동 전파되고, AI는 제안만·사람이 승인·모든 판단은 지울 수 없는 기록으로 남습니다. 그래서 노무·법령 리스크가 큰 한국 HR 현장에서 **"빠른 자동화"가 아니라 "안전한 자동화"**를 제공합니다.

---

## 1. 왜 필요한가 — 문제

한국의 중소·중견기업 HR은 세 가지 고질병을 앓습니다.

1. **반복 입력**: 같은 직원 정보를 채용·근태·급여·노무·평가에 각각 다시 입력한다. 발령 한 번에 조직도·급여·근태를 따로따로 손본다.
2. **끊기는 연결**: "연차"를 부서마다 "연가·정기휴가"로 달리 부른다. 표기가 흔들리면 집계·연동이 깨진다.
3. **사후 발견**: 주52시간 위반, 법정교육 미이수, 인증 만료를 사고가 난 뒤에야 안다.

여기에 **AI를 도입하고 싶지만** 직접 구현할 시간·기술이 없고, 노무·법령 영역이라 **"AI가 멋대로 처리하면 어쩌지"**라는 공포가 크다.

---

## 2. 무엇인가 — HR 운영 OS의 정의

AEON HR OS는 "HR 화면 묶음"이 아니라 **운영체제**입니다. 컴퓨터의 OS가 파일·프로그램·권한을 한 체계로 관리하듯, AEON은 **HR 정보·사건·업무·승인·기록**을 한 체계로 관리합니다.

핵심은 **7단계 흐름**(개발 헌법)입니다:

```
정보(코드) → 사건 → 시냅스(전파) → AI 제안 → 사람 승인 → 실행원장 → 감사기록
```

- **정보**: 유형/상태 값을 자유 문자열이 아니라 표준 숫자 코드(공통코드)로 관리 → 표기가 흔들려도 안 깨짐.
- **사건**: 신청·초과·미이수 등 HR 사건이 발생.
- **시냅스**: 관련 OS(부서)로 **코드 키로 자동 전파**.
- **AI 제안 → 사람 승인**: AI가 다음 조치 초안을 큐에 올리고, **사람이 승인해야만** 실행(반다이나믹 원칙).
- **실행원장·감사**: 무엇이 언제 어디로 갔는지 추적 + 지울 수 없는 증거.

## 3. 기능 지도 — HR 전 과정 15개 파트

> 상태: **Active**(실제 작동·검증) / **확장 중**(코어 O, 세부 진행) / **Partial**(화면·백엔드 혼재) / **[설계]**(문서·계획).

| 파트 | 무엇을 하나 | 상태 |
|---|---|---|
| **확보(채용)** | 채용계획→공고→**사람인 자동수집·이력서PDF**→AI 지원자 분석→면접→처우승인→검진/평판→최종합격→입사전환 | **Active**(가장 강함) |
| **조직·발령** | 계정·조직도·정원, 발령 승인→조직·근태·급여 자동 반영 | **Active** |
| **직무·스킬** | JD·직무등급·직무평가·Skill Library(채용·평가·보상·교육의 기준점) | **Active** |
| **근태** | 출퇴근·연차·승인요청·주52h 리스크·연차촉진·유연근무 슬롯 | **Active** |
| **노무** | 노사협의회·고충·괴롭힘·징계(사유/양정)·근로시간/임금·복합리스크 | 확장 중 |
| **보상** | 급여·수당·급여마감·4대보험·통상임금·연말정산·급여형태(직군별) | 확장 중 |
| **평가** | 평가 사이클·리뷰·성과갭 신호→교육/직무/보상 연계 | Partial |
| **교육·경력** | 교육·출장·IDP, 승인 시 근태 자동 반영 | 확장 중(백엔드) |
| **유지(관계)** | 1on1·Q12·eNPS·복리후생→이직위험 감지 | 확장 중 |
| **이직** | 퇴직·인수인계·퇴직정산·퇴직금·계정 비활성 | **Active** |
| **라이프케어** | 건강검진·모성보호·식수 | **Active** |
| **증명·컴플라이언스** | 증명서(AI 초안)·인증/기한·자동점검·알림 | **Active** |
| **HR Analytics** | 인력/이직/성과/OS 건강도·KPI·리포트 | **Active**(보고형) |
| **HR Synapse(엔진)** | 모듈 간 이벤트 연결 지도·실행 원장·Agent 승인 큐 | **Active**(전략 핵심) |
| **플랫폼(설계자)** | 공통코드 거버넌스·권한·PII·감사·QA Gate·테넌트 배포 | **Active**(내부용) |

## 4. 신뢰성 장치 — 단순 부가기능이 아니라 "운영 신뢰성"

| 장치 | 역할 |
|---|---|
| **HR Synapse** | 조직·직무·채용·근태·노무·보상·이직·컴플라이언스 사이 이벤트 자동 연결(25개 연결) |
| **Agent Action 승인 큐** | AI가 바로 실행 안 하고 → 추천 → 승인 → 실행 → 기록 |
| **SynapseRun 원장** | connection_id·source·target·status·**code_keys**로 실행 추적 |
| **Audit Log** | 민감 HR 판단·export·승인·변경 근거를 append-only로 기록 |
| **공통코드 온톨로지** | 휴가·근무·직무·평가·징계 등 값의 표준화(42그룹/221코드) |
| **QA Gate(본부실험실)** | 출시 전 기능 검증 + 테넌트 기능 ON/OFF 분리 + override 감사 |
| **Context Radar** | 코드·문서·스토리 근거 검색(챗봇 답변 근거로도 사용) |
| **PII 마스킹 / 권한 / 3모드** | 역할별 개인정보·페이지 접근 통제 |
| **Policy·Loop Engine(개념)** | 법·규정·승인 판단 / 승인된 작업만 실행·검증·기록 (일부 코드, 일부 설계 원칙) |

## 5. 기술 아키텍처

| 층 | 구성 |
|---|---|
| 프론트 | React + Vite, ~60 페이지. JWT Bearer 인증. Auth/Permissions Context. 공개경로(/, /login, /signup, /terms, /labor-qa)와 보호경로 분리. 3모드 라우팅(/hr-os, /my, /aeon-hq). |

| AI/LLM | **Google Gemini**(`gemini-2.0-flash`) 단일 게이트웨이(`chatbot/claude_ai.py`). `generate/generate_json/chat`. 키 없으면 fallback. **모델 교체 가능 구조**(벤더 종속 없음). |
| AI 에이전트 | Recruiting/Compliance/Report Agent(JD 초안·지원자 분석·리스크 보고·주간 리포트·승인 대기 액션 생성). |
| 자동화 | 사람인 이력서 수집(Selenium 기반 별도 실행). |
| CI/CD | GitHub Actions — **CI**(프론트 빌드+백엔드 단위테스트) / **CD**(프론트→Vercel, 백엔드→Railway 자동배포, 컨테이너 기동 시 DB 마이그레이션 자동). Docker(python:3.11-slim, 비루트). |
| 테스트 | 107개 테스트 함수 / 37개 파일(단위+통합). |

## 6. 무엇이 다른가 — ATS/HRIS와의 차이

- 일반 **ATS**(채용관리)는 채용만, **HRIS**(인사정보)는 저장만 한다.
- AEON은 **"HR 이벤트가 다른 업무로 전파되는 운영 OS"**다. 발령 하나가 조직·근태·급여로, 주52h가 노무로, Q12 급락이 이직위험으로 **자동으로 흐른다.**
- 그리고 그 흐름이 **AI 제안 → 사람 승인 → 불변 기록**을 관통한다 → **컴플라이언스 안전이 곧 차별점.**

---

## 7. 도입 방식 — 기존 시스템을 뺏지 않고 얹는다

타사 ERP/HRIS를 대체하려 하지 않는다. **HR 판단·자동화·증거·리스크 레이어**로 붙는다.

| 단계 | 작업 |
|---|---|
| 1 | 타사 직원/조직/직무 master를 **읽기 전용**으로 수신 |
| 2 | 고객 코드 ↔ AEON 공통코드 **매핑 테이블** 구성 |
| 3 | 채용/근태/노무/컴플라이언스 이벤트를 AEON에서 생성, 원장 변경은 **승인 후** 외부로 push |
| 4 | API/Webhook/CSV/Excel 4경로 지원 |
| 5 | Agent Action을 외부 실행 전 **승인 게이트**로 사용 |
| 6 | AuditLog·SynapseRun을 외부 전송 이력까지 확장 |
| 7 | 고객사별 Tenant Matrix로 기능 ON/OFF·권한·마스킹 분리 |

## 8. 완성도 — 있는 그대로

과장하지 않는다. 이것이 오히려 신뢰의 근거다.

- **Active(실제 작동)**: 확보·발령·근태·이직·라이프케어·컴플라이언스·Analytics·HR Synapse·플랫폼.
- **확장 중**: 노무·보상·유지·교육.
- **Partial(혼재)**: 평가, 교육/이직 일부.
- **[설계](착공 예정)**: 지식그래프·GraphRAG·온톨로지·Agent Harness(8주 후보 플랜, 1차=발령 인텔리전스 그래프).
- **주의**: MCP·플러그인·BMAD는 **개발 도구**이며 제품 런타임 기능이 아니다. 제품 AI는 Gemini 하나.

- **0~30일**: MVP 스코프 확정(Active 파트) + 완성도 100% + 데모 데이터 + E2E 증거.
- **30~60일**: 라이브 배포 확인 + 멀티테넌트 격리 + 보안/LLM 정책 + 급여밴드.
- **60~90일**: 고객 1곳 파일럿 + 피드백 + 종속 커넥터 1종(읽기연동).
- **출시 후**: 발령 인텔리전스 그래프 착공(지식그래프 1차).
