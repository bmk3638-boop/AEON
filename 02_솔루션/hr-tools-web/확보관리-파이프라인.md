# 확보관리 파이프라인 — 단계별 흐름

> 마지막 업데이트: 2026-05-10

---

## 전체 흐름도

```
인력소요 산정
    │
    ▼
채용요청서 작성 → 승인
    │
    ▼
평가기준 설정 (부서·직무별)
    │
    ▼
모집관리
├── 채용사이트 계정 관리 (사람인, 잡코리아, 링크드인...)
├── 사내공모·사내추천
├── 써치펌·헤드헌팅
├── 인턴채용 관리
├── 기간제·계약직 채용관리
├── 채용공고 관리
├── 지원자 자동 다운로드 (사람인 자동화)
└── 모집평가 (기초율/산출률/선발률/수용률 KPI)
    │
    ▼
이력서 수집 → AI 파싱 → 점수화
    │
    ▼
지원자 검토 (AcqCandidates)
├── AI 종합점수 기준 정렬
├── 상태: 면접추천 / 검토필요 / 미검토 / 탈락
└── → 면접 일정 등록 OR 최종합격 처리
    │
    ▼
면접 파이프라인 (AcqInterviews)
├── 1차 면접 → 1차 합격
├── 2차 면접 → 2차 합격
├── 3차 면접 → 3차 합격
├── 건강검진
├── 최종 합격
├── 입사일 확정 (JoinDateModal)
└── 입사자 명단
    │
    ▼
처우승인·최종합격 (AcqSelection)
├── 처우협의
├── 오퍼 발송
└── 합격 통보
    │
    ▼
입사 전환 (AcqFinalHire)
├── 신규입사 인사정보 자동 기입
└── 인사 DB 등록
    │
    ▼
입사서류 검증 (AcqOnboarding)
├── 체크리스트 (건강검진/학력증명/경력증명/계약서/보안서약)
└── 온보딩 메일 발송
```

## 핵심 데이터 흐름 (acqStore.js)

```
data.applicants      ─ 지원자 목록 (AI점수, 상태, 키워드)
data.interviews      ─ 면접 일정 (round, responseStatus, result)
data.finalPassed     ─ 최종합격자 (joinDate, checklist)
data.requisitions    ─ 채용의뢰서
data.annualPlans     ─ 연간 인력계획 (Q1~Q4)
data.mailLogs        ─ 메일 발송 이력
data.offerLogs       ─ 오퍼 발송 이력
data.channelAccounts ─ 채용사이트 계정
```

## 면접 파이프라인 상태값

### responseStatus (참석 응답)
| 값 | 표시 |
|----|------|
| `pending` | 응답 대기 |
| `accepted` | 참석 가능 |
| `change_requested` | 변경 요청 |
| `absent` | 불참 |

### result (판정 결과)
| 값 | 표시 | PipelineNav 섹션 |
|----|------|-----------------|
| `pending` | 판정 대기 | 면접 일정 |
| `first_pass` | 1차 합격 | 합격 단계 |
| `second_pass` | 2차 합격 | 합격 단계 |
| `third_pass` | 3차 합격 | 합격 단계 |
| `health_check` | 건강검진 | 입사 전환 |
| `final_pass` | 최종 합격 | 입사 전환 |
| `join_confirmed` | 입사 확정 | 입사 전환 |
| `rejected` | 불합격 | — |
| `cancelled` | 취소 | — |

## CLAUDE.md 주요 제약사항

1. `채용 자동화`를 확보관리 대표 목차로 올리지 말 것 → `모집관리` 하위 기능
2. Green Tone 유지 (`#0F8A5F`) — 오렌지 재사용 금지
3. `/acquisition` 이외 실무 화면은 `Protected` 유지
4. `sada/` 원본 폴더 수정 금지 (사람인 자동화)
5. `.env` 절대 커밋 금지
