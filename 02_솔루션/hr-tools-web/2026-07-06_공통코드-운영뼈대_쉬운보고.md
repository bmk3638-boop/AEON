# 우리 시스템 지금 무슨 상태인가 — 대표님용 쉬운 보고

## 한 줄 요약

> 지금까지 = **방(화면)들을 예쁘게 만든 단계**.
> 지금 하는 공사 = **그 방들이 서로 "같은 언어(숫자 코드)"로 대화하게 만드는 배관공사.**
> 근태(휴가·출퇴근) 방은 배관 완료. 나머지 방들은 코드표는 만들었고, 실제로 갈아끼우는 일이 남음.

---

## 1. 비유 — 우리 제품은 "자동 물류센터"를 짓는 중

우리가 만든 건 창고 안의 **방**들입니다. 채용 방, 근태 방, 노무 방, 급여 방…
문제는 방끼리 물건을 주고받을 때 **서로 다른 이름표**를 붙였다는 것.

- 근태 방: "연차"
- 급여 방: "연가"
- 노무 방: "정기휴가"

사람은 같은 말인 줄 알지만, 컴퓨터는 "이게 같은 거야?" 하고 헷갈립니다.

**그래서 하는 공사 = 모든 방이 똑같은 숫자 이름표를 쓰게 만드는 것.**
"연차"든 "연가"든 전부 **100번**. 이렇게 하면 세 가지가 좋아집니다.

1. 표기가 흔들려도 방끼리 연결이 안 끊긴다
2. 한 방에 넣으면 관련 방으로 **자동 배송**된다
3. "육아휴직(220번) 몇 건?" 같은 통계·AI 학습이 정확해진다

---

## 2. 지금 실제로 되는 것 (업무 기준)

| 되는 것 | 쉬운 설명 |
|---|---|
| **표준 코드표 생성** | 13개 업무영역에 걸쳐 192개 코드 정의됨. (연차=100, 육아휴직=220, 해고=150 …) |
| **근태 방은 완전히 코드로 작동** | 검증까지 끝남. 직원이 옛날 표기 "반차"로 신청해도 시스템이 알아서 101번으로 번역해 저장. |
| **"AI 제안 → 사람 승인 → 자동 실행 → 기록 → 감사" 라인이 살아있음** | 예: 발령을 **사람이 승인**하면 → 조직도·근태·급여로 **자동 전달** → "몇 시에 뭐가 어디로 갔다" 추적기록 + **지울 수 없는 감사기록** 남음. AI 혼자 실행 불가(노무·법령 안전장치). |

이 세 번째가 우리 제품의 핵심 차별점입니다. **AI가 멋대로 안 하고, 사람 도장 없이는 아무것도 실행 안 되며, 모든 게 지울 수 없게 기록됩니다.**

---

## 3. 아직 안 된 것

| 안 된 것 | 쉬운 설명 |
|---|---|
| **나머지 방 데이터 갈아끼우기** | 노무·급여·평가·채용 등은 코드표만 만들어놨고, 기존에 쌓인 데이터를 실제 코드로 바꾸고 검증하는 일이 남음. |
| **방끼리 배송을 "코드"로** | 지금 자동배송이 일부 구간은 아직 "글자"로 오감. 이걸 "코드"로 주고받게 바꾸는 게 마지막 배관. |
| **이름표 언어 정리(사소)** | 코드 이름표에 한글/영어가 섞여 있어 나중에 통일 필요. |

---

## 4. 왜 이게 중요한가 (Before / After)

- **Before(보통 회사):** 같은 걸 부서마다 다르게 부르니, 통계 낼 때마다 사람이 일일이 맞춰야 하고 AI가 헷갈림. 표기 하나 바뀌면 연결이 깨짐.
- **After(우리 시스템):** 원천이 코드 하나. 부서가 뭐라 부르든 뒤에선 코드로 돌아 연결이 안 깨지고, 통계·AI 학습이 자동으로 정확해짐.

---

## 5. 다음에 뭘 할지 (대표님이 고르시면 됩니다)

- **(A) 코드화 완주** — 나머지 방들 데이터를 코드로 갈아끼우고 검증.
- **(B) 자동배송 시범 전환** — 배송 하나를 코드 기준으로 바꿔 시범(예: 발령 → 근태).
- **(C) 작업지시서(스토리) 작성** — 순서대로 진행할 수 있게 문서화.
