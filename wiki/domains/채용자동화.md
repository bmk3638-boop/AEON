# 사람인 채용 자동화 파이프라인

## 아키텍처

```
사용자 (브라우저 localhost:5173)
    ↓ JWT 인증
FastAPI 백엔드 (localhost:8010)
    ↓ subprocess
pipeline.py (Selenium + Chrome)
    ↓
hiring.saramin.co.kr
    ↓
PDF 이력서 수집 → AI 스코어링 → state.json
```

## 핵심 파일

| 파일 | 역할 |
|---|---|
| `backend/modules/recruiting/pipeline.py` | 메인 자동화 스크립트 |
| `backend/modules/recruiting/router.py` | FastAPI 엔드포인트 |
| `backend/auth/users.json` | 사용자별 saramin_id/pw 저장 |
| `frontend/src/pages/Recruiting.jsx` | 실행 UI |

## 다중 사용자 지원 (2026-05-05 구현)

- 가입 시 사람인 ID/PW 함께 입력 가능
- JWT 토큰으로 유저 식별 → 해당 유저의 saramin 계정으로 로그인
- `.env`의 하드코딩 의존 제거

## 실행 흐름

1. 브라우저에서 "실행" 클릭
2. JWT 헤더로 `/api/recruiting/run` POST
3. 백엔드가 users.json에서 saramin_id/pw 조회
4. subprocess로 pipeline.py 실행 (환경변수로 자격증명 전달)
5. 로그 폴링 (`/api/recruiting/log`)

## 알려진 이슈

- Chrome `--reload` 옵션 Windows에서 다중 프로세스 문제 → `--reload` 제거
- 인증번호 SMS 필요 시 UI에서 직접 입력
