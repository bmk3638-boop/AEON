# HR Tools 확보관리 사용성 개선 2026-05-09

## 요약

2026-05-09 세션에서 `hr-tools-web`의 확보관리 MVP를 HR 컨설턴트가 실제 시연 흐름처럼 조작할 수 있게 개선했다.

기존에는 화면별 목업 데이터가 고정되어 있어 버튼을 눌러도 업무 상태가 이어지지 않았다. 이번 작업에서는 Acquisition OS 목업 상태를 브라우저 `localStorage`에 저장하고, 인력계획 → 모집흐름 → 후보자 인텔리전스 → 면접 → 선발·처우 → OCR 검토까지 핵심 상태 변경이 화면 이동 후에도 유지되도록 했다.

## 현재 실행 기준

- 현재 작업 경로: `C:\Users\qoaud\Desktop\hr-tools-web`
- 프론트 개발 서버: `http://127.0.0.1:5173`
- 백엔드 API: `http://127.0.0.1:8000`
- Vite 개발 프록시: `/api` → `http://localhost:8000`
- Docker 기준: frontend `3000`, backend `8000`

> [!warning]
> 2026-05-05 노트 일부에는 실행본 경로가 `C:\Users\qoaud\Projects\hr-tools-web`로 적혀 있다. 2026-05-09 현재 Codex 작업 컨텍스트와 실행 확인 기준은 `C:\Users\qoaud\Desktop\hr-tools-web`다.

## 구현 내용

### 상태 저장

- `frontend/src/pages/acqStore.js` 추가
- `localStorage` key: `aeon-acquisition-os-v1`
- `MOCK` 데이터를 초기값으로 사용하고, 화면 조작 후 상태를 저장
- `/acquisition` 대시보드에서 데모 초기화 가능

### 인력계획

- 채용의뢰서 생성 가능
- 승인 대기 의뢰서 승인 가능
- 보완 처리 가능
- 승인된 의뢰서에서 모집흐름으로 이동

### 모집흐름

- 작성중 공고를 모집중으로 전환 가능
- 이력서 업로드 목업 카운트 표시
- AI 파싱 시작 시 오픈 공고의 파싱 완료 수 반영

### 후보자 인텔리전스

- 후보자 상태를 추천, 보류, 탈락으로 변경 가능
- 후보자 상세 패널에서 면접 일정 등록 가능
- 최종합격 처리 시 선발·처우관리 후보자로 이동

### 면접 데스크

- 예정 면접을 합격 처리 가능
- 예정 면접을 취소/완료 처리 가능
- 대시보드의 면접 예정 수에 반영

### 선발·처우관리

- 오퍼 발송 처리 가능
- 처우 확정 저장 가능
- 검진 결과를 적합, 부적합, 보류로 판정 가능
- 입사 서류 체크리스트 상태 변경 가능

### OCR 검토

- 증빙서류 업로드 목업 추가 가능
- 불일치/확인필요 문서를 확인 완료로 변경 가능
- 대시보드의 OCR 확인 필요 수에 반영

## 검증

- `git pull`: 이미 최신
- `npm run build`: 통과
- 수정 파일 대상 ESLint: 통과
- 전체 `npm run lint`: 기존 누적 오류로 실패

## 남은 리스크

- 전체 ESLint에는 기존 파일의 React Hooks 규칙, unused 변수, Fast Refresh 관련 오류가 남아 있다.
- 실제 백엔드 영속 저장이 아니라 프론트 로컬 데모 상태다.
- `CLAUDE.md`, `OBSIDIAN_HUB.md`, 과거 AEON 노트 사이의 실행 경로 표기가 섞여 있다.
- `docs/`는 `hr-tools-web` 레포 내부 문서이고, 실제 Obsidian 볼트는 `C:\Users\qoaud\AEON`이다.
