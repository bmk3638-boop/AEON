# 확보관리 파이프라인 복구 2026-05-05

## 반드시 기억할 것

확보관리는 단순 채용 화면이 아니다. 다음 순서가 제품의 핵심 구현 흐름이다.

1. 공고 스캔
2. 사람인 이력서 자동 다운로드
3. PDF 과학적 파싱
4. OCR 서류 교차검토
5. 지원자 평가·면접 파이프라인
6. 신규입사등록·인사정보등록

이 흐름은 삭제하거나 축소하지 않는다. 디자인을 정리하더라도 위 단계가 화면에서 보이도록 유지한다.

## 인증번호 반복 팝업 디버그

**Symptom:** 확보관리 진입 시 사람인 인증번호 모달이 계속 뜸.

**Root cause:** `fetchLog()`가 과거 로그에 남아 있던 `WEB_AUTH` 또는 `인증번호 입력 대기` 줄을 새 인증 요청으로 오해했다. 페이지 진입 때 기존 로그를 읽자마자 `setShowAuth(true)`가 실행됐다.

**Fix:** 최초 로그 읽기 시 인증 줄 수를 기준선으로 저장한다. 자동화가 실행 중이고, 기준선 이후 새 인증 줄이 생길 때만 인증번호 모달을 연다.

**Evidence:** `npm run build` 성공, 핵심 파일 ESLint 성공, 검색으로 `AskHR`와 이전 검은 로그 문구 제거 확인.

## 다음 작업자 주의

확보관리 UI를 다시 다듬을 때는 “검은 로그 터미널 제거”와 “자동화 프로세스 제거”를 혼동하지 말 것. 서버 로그는 사용자에게 직접 노출하지 않되, 제품 기능 흐름은 반드시 명확하게 보여준다.
