# 멀티유저 아키텍처

## 핵심 정의

하나의 플랫폼에서 여러 사용자가 각자의 외부 서비스 자격증명을 독립적으로 사용하는 구조.

## hr-tools-web 구현 방식

```
회원가입 시
  └── email + password + saramin_id + saramin_pw 입력
      └── users.json에 사용자별 저장

자동화 실행 시
  └── JWT 토큰으로 사용자 식별
      └── users.json에서 해당 유저 saramin_id/pw 조회
          └── subprocess 환경변수로 pipeline.py에 전달
```

## 핵심 파일

| 파일 | 역할 |
|---|---|
| `backend/auth/users.json` | 사용자별 saramin_id/pw 저장 |
| `backend/auth/router.py` | 가입 시 saramin 정보 저장 |
| `backend/modules/recruiting/router.py` | JWT → saramin 자격증명 조회 |

## 보안 고려사항

- saramin 비밀번호는 평문 저장 (외부 서비스 특성상 해시 불가)
- users.json은 서버 내부에서만 접근 — 절대 API로 노출 금지
- JWT 만료 시간 관리 필요
