## #2 진단 — 테넌트 QA게이트 "작동 안함"

- 증상: 테넌트 기능 OFF 후 다시 ON 안 됨. "API가 죽었나?"
- 진단(증거): 백엔드 `/docs` 200(정상). enable → **400 "Latest QA run is not release-ready"**, disable → 200.
- 결론: **API 문제 아님. B1/B2로 게이트가 정직해지며 release_ready가 거의 다 False(데모가 실검사 통과 못 함)** → "QA 미통과 기능은 켤 수 없다"고 게이트가 올바르게 막는 것. 예전엔 가짜 점수라 항상 켜졌을 뿐.
- 제품으론 옳지만 데모엔 불편 → **D3 후보: 설계자 사유기록 override**(QA 미통과여도 설계자가 사유 남기고 활성, 로그 보존 — 반다이나믹 원칙).

## 실수 & 보완 (이번 세션) → 누적: [[실수_보완_누적]]

1. **거짓 모지바케 함정**: `curl | python -m json.tool`로 API를 보니 한글이 깨져(`\udcec...`) 인코딩 버그로 오인할 뻔. 원인은 Windows python stdin이 cp949로 UTF-8을 잘못 디코딩한 **파이프 아티팩트**. → hexdump로 원본 바이트 확인하니 정상 UTF-8. **교훈: 인코딩 의심 시 원본 바이트로 검증, 결과는 파일로 떠서 Read.**
2. **도구 호출 형식 오류(재발)**: 일부 도구 호출을 잘못된 태그로 작성해 파싱 실패. **교훈: 정확한 호출 형식 엄수.**
3. **코드만 읽고 '양호' 단정 금지**: 과거 설계자모드를 grep만 보고 "양호"라 했다가 실제 화면이 엉망이었음. 이번엔 빌드+API 스모크+수치로 검증. **단, 이 환경은 WDAC 정책으로 브라우저 바이너리 실행 차단 → 스크린샷 불가 → API/수치/빌드로 대체.**
4. **정직화의 부수효과 미리 점검**: QA를 정직하게 만들자 테넌트 enable이 막히는 연쇄가 발생. **교훈: 게이트/판정 로직을 바꿀 땐 그 값에 의존하는 다운스트림 플로우(여기선 테넌트 활성)도 같이 점검·고지.**
5. **경로 스타일 함정**: MSYS 경로(`/c/...`)를 Windows python/curl에 넘겨 FileNotFound. → Windows 경로(`C:/...`)로 통일.
