### 2. MCP 서버 3개 연결 실패

  - claude.ai 클라우드 서버들만 로드됨
- **해결**: `C:\Users\bmk36\.claude\.mcp.json` 생성 (MCP 서버 3개 정의)
- **상태**: 🔄 VSCode 재시작 후 확인 필요

- **상태**: ⚠️ VSCode 익스텐션 한계 — 근본 해결 불가

### 5. Docker 미설치 → Harness 실행 불가

## 헤르메스(Hermes)와의 협업 구상

### 헤르메스란?
그리스 신화에서 신들의 메신저. AEON 오케스트레이션에서:
- **역할**: 에이전트 간 메시지/태스크 라우팅 레이어
- **AGT-00(Claude)이 실행 뇌**라면, 헤르메스는 **통신 신경망**

### 현재 오케스트레이션 구조 문제
```
배명규 → Claude(AGT-00) → 에이전트들
```
- 단방향, Claude가 직접 모든 것을 실행
- 에이전트 간 병렬 실행/상태 추적 없음
- 결과가 Claude 컨텍스트에만 존재, Obsidian에 자동 저장 안 됨

### 헤르메스 도입 후 구조
```
배명규
  → Claude(AGT-00, 실행 뇌)
      → Hermes(메신저/라우터)
          ├─ AGT-16(사람인) → 결과 → Obsidian 자동 기록
          ├─ AGT-17(이력서) → 결과 → Obsidian 자동 기록
          ├─ AGT-18(대시보드) → 결과 → 파일 저장
          └─ AGT-20(Obsidian관리자) → 볼트 동기화
```

### 헤르메스 구현 후보

| 방법 | 도구 | 특징 |
| --- | --- | --- |
| 즉시 가능 | n8n 워크플로우 | 노코드, AEON 기존 스택 |
| 단기 | Claude Subagents (Agent 도구) | 이미 Claude Code에 내장 |
| 중기 | VoltAgent | TypeScript, 모니터링 포함 |

### 지금 당장 쓸 수 있는 것: Claude Subagent
Claude Code의 `Agent` 도구가 이미 헤르메스 역할 가능:
```
AGT-00(Claude) → Agent(subagent) 병렬 실행
  ├─ Subagent A: 사람인 스크래핑
  ├─ Subagent B: PDF 파싱
  └─ Subagent C: 대시보드 생성
결과 → 자동으로 Obsidian에 기록
```

### 오케스트레이션 원칙
1. **병렬 실행**: 독립적인 작업은 동시에 처리
2. **결과 즉시 저장**: 모든 작업 결과는 Obsidian에 자동 기록
3. **상태 추적**: `05_운영/세션로그/`에 작업 진행 상황 실시간 업데이트
4. **실패 복구**: 에이전트 실패 시 대체 경로 존재

## Claude-Obsidian 관계 정의

| 역할 | 담당 | 설명 |
| --- | --- | --- |
| 행동하는 몸 | Claude(AGT-00) | 코드 실행, 웹 검색, 파일 조작 |
| 실행 뇌 | Claude(AGT-00) | 판단, 계획, 오케스트레이션 |
| 기억하는 뇌 | Obsidian | 지식 저장, 컨텍스트 유지, 누적 학습 |
| 통신 신경망 | Hermes(n8n/VoltAgent) | 에이전트 간 메시지 라우팅 |

**핵심 원칙**: Claude가 배운 것 = AEON이 배운 것 = 즉시 Obsidian에 기록
